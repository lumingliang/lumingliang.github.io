php + redis 处理高并发

利用队列处理高并发，就像排队一样，其实和锁是一样(?)，但是队列更容易理解，队列是原子操作

  1. 将库存加入队列，每下一个单前pop一个值，能pop说明库存还有
  2. 分三个队列，等待用户，已购买用户，库存，开一个进程不断从等待用户中取出并判断是否在已购买队列中，再决定是否下单，减少库存
  3. 从库存队列中pop出后，实际上还需要在数据库中对库存减1，如果其中出错就需要对队列+1
